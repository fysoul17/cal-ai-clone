## User Scenarios & Testing *(mandatory)*

### User Story 1 - First Impression & Value Understanding (Priority: P1)

신규 방문자가 랜딩 페이지에 접속하여 Cal AI Web Clone이 무엇인지, 어떤 가치를 제공하는지 즉시 이해하고 서비스 사용을 결심한다.

**Why this priority**: 랜딩 페이지의 핵심 목적은 방문자를 사용자로 전환하는 것이다. 제품의 핵심 가치(사진만 찍으면 칼로리/영양소 자동 분석)를 명확하게 전달하지 못하면 이탈률이 높아진다.

**Independent Test**: 방문자가 페이지 로드 후 5초 이내에 "이 서비스가 무엇을 하는지" 설명할 수 있어야 한다. 히어로 섹션만 구현해도 MVP로서 가치를 전달할 수 있다.

**Acceptance Scenarios**:

1. **Given** 사용자가 처음 랜딩 페이지에 접속, **When** 페이지가 로드됨, **Then** 다크 테마 배경에 프리미엄한 히어로 섹션이 표시되고 "사진 한 장으로 칼로리 추적" 핵심 메시지와 앱 스크린샷/목업이 함께 보인다
2. **Given** 히어로 섹션이 표시됨, **When** 사용자가 CTA 버튼을 확인, **Then** "지금 시작하기" 또는 유사한 행동 유도 버튼이 눈에 띄게 표시된다
3. **Given** 페이지가 로드됨, **When** 애니메이션이 실행됨, **Then** 텍스트와 요소들이 부드럽게 페이드인되며 시각적 흥미를 유발한다

---

### User Story 2 - Feature Discovery (Priority: P2)

방문자가 스크롤하면서 Cal AI의 주요 기능들(AI 음식 분석, 자동 영양소 계산, 일일 추적)을 상세히 파악한다.

**Why this priority**: 핵심 가치를 이해한 후, 구체적인 기능을 통해 신뢰를 구축하고 가입 결정을 강화한다.

**Independent Test**: 기능 섹션만 구현해도 제품의 차별점을 전달할 수 있다. 각 기능 카드를 독립적으로 테스트 가능하다.

**Acceptance Scenarios**:

1. **Given** 사용자가 히어로 섹션 아래로 스크롤, **When** 기능 섹션에 도달, **Then** 3-4개의 주요 기능이 카드 형태로 시각적으로 표현된다
2. **Given** 기능 카드가 화면에 보임, **When** 스크롤하여 뷰포트에 진입, **Then** 카드가 부드러운 애니메이션과 함께 나타난다
3. **Given** 각 기능 카드, **When** 사용자가 확인, **Then** 아이콘, 제목, 간단한 설명이 포함되어 기능을 직관적으로 이해할 수 있다

---

### User Story 3 - How It Works Flow (Priority: P2)

방문자가 Cal AI 사용 방법을 단계별로 이해하여 서비스가 얼마나 간단한지 확인한다.

**Why this priority**: 복잡해 보이는 AI 기술을 쉽게 사용할 수 있다는 것을 시각적으로 보여줌으로써 진입 장벽을 낮춘다.

**Independent Test**: 사용 방법 섹션만으로도 사용자 온보딩의 예비 단계 역할을 수행할 수 있다.

**Acceptance Scenarios**:

1. **Given** 사용자가 "사용 방법" 섹션에 도달, **When** 섹션이 표시됨, **Then** 3단계 프로세스(사진 촬영 → AI 분석 → 결과 저장)가 시각적으로 표현된다
2. **Given** 단계별 안내, **When** 사용자가 스크롤, **Then** 각 단계가 순차적인 애니메이션으로 나타난다
3. **Given** 각 단계, **When** 사용자가 확인, **Then** 단계 번호, 제목, 간단한 설명, 관련 아이콘/일러스트가 포함된다

---

### User Story 4 - Trust Building & CTA (Priority: P3)

방문자가 페이지 하단에서 마지막 행동 유도 섹션을 통해 회원가입/로그인 페이지로 이동한다.

**Why this priority**: 모든 정보를 확인한 사용자에게 마지막 전환 기회를 제공한다.

**Independent Test**: 하단 CTA 섹션만으로도 전환 포인트로서 기능한다.

**Acceptance Scenarios**:

1. **Given** 사용자가 페이지 하단에 도달, **When** CTA 섹션이 표시됨, **Then** 강렬한 행동 유도 메시지와 버튼이 표시된다
2. **Given** CTA 버튼, **When** 사용자가 클릭, **Then** 회원가입/로그인 페이지로 이동한다
3. **Given** 하단 섹션, **When** 표시됨, **Then** 푸터에 필수 정보(저작권, 링크 등)가 포함된다

---

### Edge Cases

- 저사양 기기나 느린 네트워크에서 접속 시 어떻게 처리하는가? → 애니메이션 자동 축소 및 이미지 최적화 (prefers-reduced-motion 미디어 쿼리 적용)
- 스크린 리더 사용자가 접근 시 어떻게 처리하는가? → 모든 이미지에 대체 텍스트 제공, 의미있는 HTML 구조 사용
- 매우 긴 텍스트가 다국어로 표시될 경우 레이아웃이 어떻게 유지되는가? → 현재 한국어 단일 언어 지원 (향후 확장 가능)
- 사용자가 시스템 라이트 모드를 선호하는 경우 어떻게 처리하는가? → 랜딩 페이지는 다크모드로 고정 (브랜드 아이덴티티)

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템 MUST 다크 테마를 기본으로 하는 랜딩 페이지를 표시한다
- **FR-002**: 시스템 MUST 히어로 섹션에 제품의 핵심 가치 메시지("사진 한 장으로 칼로리 추적")와 앱 스크린샷/목업을 표시한다
- **FR-003**: 시스템 MUST 눈에 띄는 CTA(Call-to-Action) 버튼을 제공하여 사용자를 가입/로그인 페이지로 안내한다
- **FR-004**: 시스템 MUST 주요 기능(AI 음식 분석, 자동 영양소 계산, 일일 추적)을 소개하는 기능 섹션을 포함한다
- **FR-005**: 시스템 MUST 서비스 사용 방법을 3단계(사진 촬영 → AI 분석 → 결과 저장)로 설명하는 섹션을 포함한다
- **FR-006**: 시스템 MUST 스크롤 시 요소들이 부드럽게 나타나는 애니메이션을 적용한다
- **FR-007**: 시스템 MUST 모바일(320px 이상), 태블릿, 데스크톱(1920px 이상)에서 반응형으로 동작한다
- **FR-008**: 시스템 MUST 페이지 로드 시 히어로 섹션 요소들이 순차적으로 페이드인된다
- **FR-009**: 시스템 MUST 네비게이션 헤더를 포함하여 로고와 CTA 버튼을 항상 접근 가능하게 한다
- **FR-010**: 시스템 MUST 푸터에 저작권 정보를 포함한다
- **FR-011**: 시스템 MUST 모션 감소 설정(prefers-reduced-motion)을 존중하여 애니메이션을 비활성화한다
- **FR-012**: 시스템 MUST 모든 이미지에 대체 텍스트를 제공한다

### Key Entities

- **Landing Page Section**: 히어로, 기능, 사용방법, CTA, 푸터 등 페이지를 구성하는 독립적인 영역
- **Call-to-Action (CTA)**: 사용자 전환을 유도하는 버튼 또는 링크 요소
- **Feature Card**: 제품의 주요 기능을 설명하는 시각적 컴포넌트 (아이콘, 제목, 설명 포함)
- **Step Indicator**: 사용 방법을 순서대로 보여주는 단계 표시 요소

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 페이지 로드 완료 후 3초 이내에 모든 초기 애니메이션이 완료된다
- **SC-002**: 모바일 기기(320px 이상)에서 데스크톱(1920px 이상)까지 레이아웃이 깨지지 않고 표시된다
- **SC-003**: 방문자의 70% 이상이 히어로 섹션 아래로 스크롤한다 (참여도 지표)
- **SC-004**: CTA 버튼 클릭률 5% 이상 달성 (전환율 지표)
- **SC-005**: 페이지 전체가 접근성 기준(WCAG 2.1 AA)을 충족한다
- **SC-006**: 저사양 기기에서도 스크롤 시 부드러운 성능(초당 30프레임 이상)을 유지한다

## Assumptions

- 사용자는 주로 모바일 기기에서 접속할 것으로 예상 (모바일 퍼스트 디자인)
- 브랜드 색상은 PRD에 언급된 "프리미엄한 다크 테마"를 따르며, 액센트 색상은 오렌지/앰버 계열을 사용
- 애니메이션은 성능에 영향을 주지 않는 범위 내에서 적용
- 텍스트 콘텐츠는 한국어로 작성
- CTA 버튼은 현재 /login 또는 /signup 경로로 연결 (실제 인증 페이지는 별도 구현)
- 이미지/일러스트 에셋은 구현 시 제작 또는 적절한 대체물 사용

## Out of Scope

- 다국어 지원 (향후 확장)
- 라이트 모드 토글 기능 (랜딩 페이지는 다크모드 고정)
- 사용자 리뷰/후기 섹션 (MVP 이후)
- 가격 정책 섹션 (PRD에서 구독 모델 미확정)
- 블로그/콘텐츠 마케팅 섹션
- 실시간 데모 기능
